# Brainstorming Session Results

## Session Overview

**Topic:** タイムボクシング式タスク管理Webアプリ
**Goals:** 日々のExcel手動入力の自動化、タグ別月次集計、レポート出力の効率化

### 背景・課題

- 一日に複数タスクをこなすが、どの時間帯に何をやったかの把握が難しい
- 毎日それぞれの稼働時間をExcelにまとめて提出する必要がある
- タイムボクシングで可視化することで計算・記述の手間を減らしたい

### Session Setup

- 想定ユーザー: 個人利用（チーム管理は拡張機能として後期対応）
- コアの痛み: 事後にExcelで稼働時間を思い出して書く作業
- 解決アプローチ: 計画編集型（タイマー型ではない）— 計画を立てて、ズレたら直す

## Technique Selection

**Approach:** AI-Recommended Techniques

**実施テクニック:**

- **Mind Mapping:** プロジェクト全体像の構造化と要素間の関係性整理
- **SCAMPER Method:** 既存Excel管理を7つの視点で再考し、機能のスコーピングと優先度決定

## Technique Execution Results

### Mind Mapping — 全体構造の整理

- コア機能と拡張機能の切り分けが明確化
- 人材管理はチーム作成フローの複雑さから拡張機能に分離
- 計画編集型のUXコンセプトが確立（タイマー型ではない）
- タスクの「完了」と時間の「確定」が独立した概念であることを発見
- 1つのTodoが複数日にまたがるケースへの対応設計

### SCAMPER Method — 機能の精査と革新

- **Substitute:** Googleカレンダー連携は拡張機能に。タグ自動推測は不採用（ユーザー手動選択）。レポートは手動ダウンロード
- **Combine:** 3カラム→2カラム構成の発見。「今日の予定にする」ボタン式UIの確立
- **Adapt:** カレンダー式縦時間軸UIの採用。グラフ可視化は拡張機能に
- **Modify:** 15分単位の粒度確定。休憩トグル採用。プロジェクト別グルーピング採用
- **Eliminate:** 実績カラムの削除→1カラムUI（データは計画/実績の2つ保持）
- **Reverse:** 計画は必須ではない設計。時刻ベースの計画/実績自動判定ロジック

## Idea Organization and Prioritization

### テーマ1: UIアーキテクチャ

- **2カラム構成** — Todoリスト + タイムライン
- **カレンダー式縦時間軸** — ブロックの高さ＝時間の長さ
- **プロジェクト別Todoグルーピング** — 折りたたみ式
- **プロジェクト配下の「+」アイコン** — タスク追加（プロジェクトタグ自動付与）
- **一番下に「+ 新しいプロジェクトを追加」ボタン** — プロジェクト作成は別導線
- **「未分類」グループ** — プロジェクト未定のタスク用
- **「今日の予定にする」ボタン** → 開始/終了時間選択 → タイムラインに反映
- **休憩トグルボタン**

### テーマ2: データモデル・ロジック

- **計画/実績の自動判定** — 追加時間枠 < 現在時刻 → 実績、≥ 現在時刻 → 計画
- **時間経過で計画→実績に自動変換** — 編集されなければそのまま実績に、編集されたら差分記録
- **15分単位の時間粒度**
- **1つのTodoに対して「タスク完了」と「時間確定」の独立した状態管理**
- **計画は必須ではない** — 後から実績だけ入力も可能

### テーマ3: タグ・分類システム

- **プロジェクトタグ** — ユーザーが自由に作成・管理
- **作業種別タグ** — ユーザーが自由に作成・管理（設計、実装、会議、レビュー等）
- **プロジェクト配下でタスク作成 → プロジェクトタグ自動付与**
- **作業種別はユーザーが手動選択**

### テーマ4: レポート・可視化

- **月次ダッシュボード** — タグ別集計 + 計画vs実績の差分分析
- **Excel出力** — プロジェクト×月の提出用（フォーマット固定）
- **PDF出力** — 振り返りレポート
- **手動ダウンロード（ボタン式）**

### テーマ5: 拡張機能（後期）

- Googleカレンダー連携（会議の自動取り込み）
- チーム作成・人材管理（マネージャーがチーム全体を俯瞰）
- パターン学習テンプレート（曜日ごとの定例タスク自動配置）
- グラフ可視化（ドーナツ/棒グラフ）
- 時間枠先行モード
- 工数見積もりへの転用
